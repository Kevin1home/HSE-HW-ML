# Wine Classification (Red vs White)

Проект посвящён бинарной классификации вина (красное vs белое) на основе физико-химических характеристик.
Работа выполнена в рамках домашнего задания по машинному обучению.

---

# Цель работы

Закрепить полный пайплайн классического ML:

* загрузка данных через Hugging Face Datasets
* разведочный анализ данных (EDA)
* предобработка признаков
* обучение нескольких моделей
* оценка качества и анализ результатов

---

# Датасет

Использован датасет:
`mstz/wine` (Hugging Face Datasets)

Описание:

* Табличные данные о винах
* Числовые физико-химические признаки
* Бинарная целевая переменная

## Задача

Предсказать тип вина:

* 1 — красное
* 0 — белое

---

# EDA (Exploratory Data Analysis)

Проведён базовый анализ данных:

* Размер датасета и структура
* Типы признаков
* Проверка пропусков
* Распределение классов
* Визуализация признаков

Основные наблюдения:

* Пропусков нет
* Все признаки числовые
* Некоторые признаки хорошо разделяют классы (например, alcohol и density)

---

# Предобработка данных

Использован `sklearn Pipeline`, включающий:

* StandardScaler для масштабирования признаков
* Модель классификации внутри Pipeline

Это обеспечивает:

* отсутствие утечек данных
* корректную обработку train/test split

---

# Модели

## 1. Logistic Regression

Базовая линейная модель, использована как baseline.

Преимущества:

* простота
* интерпретируемость
* быстрый baseline

---

## 2. K-Nearest Neighbors (KNN)

Нелинейная модель на основе расстояний между объектами.

Проводились эксперименты с параметром:

* количество соседей (k)

Цель — сравнить линейную и нелинейную модели.

---

# Оценка качества

Использованные метрики:

* Accuracy
* F1-score
* Classification report

Также проводился анализ различий между моделями.

---

# Эксперименты

## Изменение гиперпараметров

Для KNN изменялся параметр k.
Это позволило оценить влияние сложности модели на качество.

---

## Удаление признаков

Проводились эксперименты с исключением отдельных признаков, чтобы понять их вклад в качество модели.

Вывод:

* Некоторые признаки существенно влияют на разделимость классов
* Удаление ключевых фич снижает точность

---

## Кросс-валидация

Добавлена кросс-валидация для проверки устойчивости результатов.

Вывод:

* Метрики стабильны
* Сильной зависимости от случайного разбиения нет

---

# Итоговые выводы

* Logistic Regression даёт сильный baseline благодаря хорошей разделимости данных
* KNN позволяет проверить нелинейные зависимости и влияние гиперпараметров
* Датасет хорошо подходит для учебных задач табличного ML
* Признаки типа density и alcohol играют важную роль в классификации
